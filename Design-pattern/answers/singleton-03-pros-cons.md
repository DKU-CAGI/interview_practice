## Q. 싱글톤 패턴의 장점과 단점은 무엇인가요?

### 핵심 답변
싱글톤은 인스턴스 공유로 메모리 효율과 일관성을 제공하지만, 전역 상태와 강한 결합으로 인해 테스트와 유지보수가 어렵습니다.

### 상세 설명

**장점**
| 장점 | 설명 |
|------|------|
| 인스턴스 중복 생성 방지 | 하나의 인스턴스만 유지하여 메모리 절약 |
| 전역 접근 | 어디서든 동일한 인스턴스에 접근 가능 |
| 초기화 비용 절감 | DB 연결 등 무거운 객체를 한 번만 생성 |
| 일관된 상태 | 공유 자원의 상태가 항상 일관됨 |

**단점**
| 단점 | 설명 |
|------|------|
| 전역 상태(Global State) | 코드 어디서든 변경 가능해 예측 불가능한 버그 유발 |
| 단위 테스트 어려움 | 테스트 간 상태 공유로 격리 불가 |
| 강한 결합(Tight Coupling) | 싱글톤에 의존하는 코드가 강하게 결합됨 |
| 멀티스레드 위험 | 동기화 없이 Race Condition 발생 가능 |
| SRP 위반 가능성 | 인스턴스 관리 + 비즈니스 로직을 동시에 담당 |

**안티패턴으로 불리는 이유:**
- OCP 위반: 싱글톤 내부 변경이 이를 사용하는 모든 코드에 영향
- DIP 위반: 구체 클래스(싱글톤)에 직접 의존
- 의존성 주입(DI)으로 대부분의 단점을 해결할 수 있음

### 장단점
- **장점**: 메모리 절약, 일관성 보장, 공유 자원 관리 용이
- **단점**: 전역 상태 위험, 테스트 어려움, SOLID 원칙 위반 가능성

### 꼬리 질문
- 싱글톤 대신 의존성 주입(DI)을 사용하면 어떤 이점이 있나요?
- SOLID 원칙 관점에서 싱글톤이 안티패턴인 이유를 설명해주세요.

### 실무 사례
Spring Framework는 IoC 컨테이너가 싱글톤 빈을 관리함으로써 단점(테스트 어려움, 강한 결합)을 보완합니다. 컨테이너가 의존성을 주입해주므로 테스트 시 mock으로 교체할 수 있습니다.
