## Q. 프록시 서버로 nginx와 CloudFlare 중 무엇을 선택할지 어떻게 결정하나요?

### 핵심 답변
nginx는 자체 서버 환경에서 세밀한 설정이 필요할 때, CloudFlare는 CDN, DDoS 방어, 전 세계 분산 엣지 네트워크가 필요할 때 선택합니다. 두 가지를 함께 사용하는 것도 일반적입니다.

### 상세 설명

**nginx 선택 기준:**
| 상황 | nginx |
|------|-------|
| 자체 서버(온프레미스, 개인 클라우드) | ✅ |
| 로드 밸런싱 세밀한 설정 | ✅ |
| SSL 종료 + 내부 HTTP | ✅ |
| 정적 파일 직접 서빙 | ✅ |
| 비용이 없어야 함 | ✅ |
| 설정 완전한 제어 필요 | ✅ |

**CloudFlare 선택 기준:**
| 상황 | CloudFlare |
|------|-----------|
| 전 세계 사용자 (CDN 필요) | ✅ |
| DDoS 방어 필요 | ✅ |
| DNS 관리 통합 | ✅ |
| 서버 IP 숨김 필요 | ✅ |
| 빠른 설정, 적은 운영 공수 | ✅ |
| WAF(웹 방화벽) 필요 | ✅ |

**두 가지 차이점:**
| | nginx | CloudFlare |
|---|---|---|
| 위치 | 자체 서버 | 엣지 네트워크 (전 세계) |
| 관리 | 직접 설정/운영 | 웹 대시보드 |
| CDN | 별도 구성 필요 | 기본 제공 |
| DDoS | 제한적 | 강력 |
| 비용 | 서버 비용만 | Free~기업 플랜 |

**함께 사용하는 아키텍처:**
```
사용자 → CloudFlare (CDN, DDoS, SSL) → nginx (내부 라우팅) → 앱 서버
```
- CloudFlare: 외부 트래픽 처리, 글로벌 CDN
- nginx: 내부 로드 밸런싱, 정적 파일 서빙

### 꼬리 질문
- CloudFlare의 무료 플랜으로 충분한 경우는 언제인가요?
- nginx와 CloudFlare를 함께 사용할 때 주의할 점은?

### 실무 사례
스타트업은 CloudFlare 무료 플랜 + 단일 서버 nginx로 시작하여 트래픽이 증가하면 CloudFlare 엔터프라이즈 + nginx 클러스터로 확장하는 패턴을 많이 사용합니다.
