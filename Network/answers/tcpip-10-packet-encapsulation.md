## Q. 캡슐화/비캡슐화 과정과 PDU를 계층별로 설명하시오.

### 핵심 답변

**PDU(Protocol Data Unit)**: 네트워크의 어떠한 계층에서 계층으로 데이터가 전달될 때 한 덩어리의 단위

| 계층 | PDU 명칭 |
|------|---------|
| 애플리케이션 계층 | 메시지 |
| 전송 계층 | 세그먼트(TCP), 데이터그램(UDP) |
| 인터넷 계층 | 패킷 |
| 링크 계층 | 프레임(데이터 링크 계층), 비트(물리 계층) |

### 보충 설명

**캡슐화 과정 (송신 측, 위→아래)**:
```
[데이터]                     ← 애플리케이션 계층
[TCP(L4) 헤더 | 데이터]      ← 전송 계층 (세그먼트화)
[IP(L3) 헤더 | TCP 헤더 | 데이터]  ← 인터넷 계층 (패킷화)
[프레임 헤더 | IP 헤더 | TCP 헤더 | 데이터 | 프레임 트레일러]  ← 링크 계층 (프레임화)
```
상위 계층의 헤더와 데이터를 하위 계층의 데이터 부분에 포함시키고, 해당 계층의 헤더를 삽입하는 과정

**비캡슐화 과정 (수신 측, 아래→위)**:
하위 계층에서 상위 계층으로 올라가면서 각 계층의 헤더 부분을 제거하는 과정

**이더넷 프레임 구조**:
- Preamble: 이더넷 프레임의 시작 알림
- SFD(Start Frame Delimiter): 다음 바이트부터 MAC 주소 필드 시작
- DMAC/SMAC: 수신/송신 MAC 주소
- EtherType: IPv4 또는 IPv6
- Payload: 전달받은 데이터
- CRC: 에러 확인 비트

### 키워드
- PDU, 캡슐화, 비캡슐화, 세그먼트, 패킷, 프레임, 이더넷 프레임
