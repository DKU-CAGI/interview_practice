## Q. HTTP/2의 특징(멀티플렉싱, 헤더 압축, 서버 푸시)을 설명하시오.

### 핵심 답변

**HTTP/2**는 SPDY 프로토콜에서 파생된 HTTP/1.x보다 지연 시간을 줄이고 응답 시간을 더 빠르게 할 수 있는 프로토콜입니다.

**주요 특징:**

1. **멀티플렉싱(Multiplexing)**
   - 여러 개의 스트림(stream)을 사용하여 송수신
   - 단일 연결을 사용하여 병렬로 여러 요청을 받고 응답 가능
   - 특정 스트림의 패킷이 손실되어도 해당 스트림에만 영향, 나머지 스트림은 정상 동작
   - HTTP/1.x의 HOL Blocking 문제 해결

2. **헤더 압축(Header Compression)**
   - HTTP/1.x의 크기가 큰 헤더 문제를 해결
   - **허프만 코딩(Huffman Coding)** 압축 알고리즘을 사용하는 **HPACK** 압축 형식
   - 허프만 코딩: 빈도가 높은 정보는 적은 비트 수로, 빈도가 낮은 정보는 많은 비트 수로 표현

3. **서버 푸시(Server Push)**
   - HTTP/1.1에서는 클라이언트가 서버에 요청을 해야 파일을 다운로드 받을 수 있었음
   - HTTP/2는 **클라이언트 요청 없이 서버가 바로 리소스를 푸시** 가능
   - 예: html을 요청하면 그 안에 포함된 css 파일을 서버에서 먼저 클라이언트에 푸시

### 보충 설명

**HTTP/2 vs HTTP/1.1 연결 비교:**
```
HTTP/1.1: 3 TCP 연결 → example.js / example.css / image.png 각각 연결
HTTP/2:   1 TCP 연결 → example.js + example.css + image.png 동시 처리
```

**스트림(stream)**: 시간이 지남에 따라 사용할 수 있게 되는 일련의 데이터 요소를 가리키는 데이터 흐름

**HTTP/2의 한계**: TCP 위에서 동작하므로 TCP 수준의 HOL Blocking은 여전히 존재 → HTTP/3에서 해결
