## Q. L2 스위치가 MAC 주소 테이블을 학습하는 과정을 설명하시오.

### 핵심 답변

L2 스위치는 장치들의 **MAC 주소를 MAC 주소 테이블을 통해 관리**하며, 연결된 장치로부터 패킷이 왔을 때 패킷 전송을 담당합니다.

**동작 원리:**
1. 패킷이 들어오면 출발지 MAC 주소를 테이블에 기록
2. 목적지 MAC 주소가 테이블에 **있으면** → 해당 포트로만 전송
3. 목적지 MAC 주소가 테이블에 **없으면** → 전체 포트에 전달 **(플러딩, flooding)**
4. MAC 주소 테이블의 주소는 **일정 시간 이후 삭제**

### 보충 설명

L2 스위치의 특징:
- IP 주소를 이해하지 못하므로 **IP 주소 기반 라우팅은 불가능**
- 단순히 패킷의 MAC 주소를 읽어 스위칭하는 역할
- **브리지(bridge)**: 두 개의 근거리 통신망(LAN)을 상호 접속할 수 있도록 하는 통신망 연결 장치, MAC 주소 테이블로 관리

| 구분 | L2 스위치 | L3 스위치 |
|------|----------|----------|
| 참조 테이블 | MAC 주소 테이블 | 라우팅 테이블 |
| 참조 PDU | 이더넷 프레임 | IP 패킷 |
| 참조 주소 | MAC 주소 | IP 주소 |
