## Q. 트리, 버스, 스타, 링형 토폴로지의 장단점을 각각 설명하시오.

### 핵심 답변

네트워크 토폴로지(network topology)란 **노드와 링크가 어떻게 배치되어 있는지에 대한 방식이자 연결 형태**입니다.

| 토폴로지 | 구조 | 장점 | 단점 |
|---------|------|------|------|
| **트리(tree)** | 계층형, 루트 노드 중심 | 노드 추가/삭제 쉬움 | 특정 노드에 트래픽 집중 시 하위 노드 영향 |
| **버스(bus)** | 중앙 통신 회선 하나에 여러 노드 | 설치 비용 적음, 신뢰성 우수, 노드 추가/삭제 쉬움 | **스푸핑** 가능 |
| **스타(star)** | 중앙 노드에 모두 연결 | 노드 추가/에러 탐지 쉬움, 패킷 충돌 적음, 장애 노드가 다른 노드에 영향 적음 | **중앙 노드 장애 시 전체 사용 불가**, 설치 비용 고가 |
| **링형(ring)** | 각 노드가 양 옆 두 노드와 연결, 고리 형태 | 노드 수 증가해도 손실 적음, 충돌 가능성 적음 | 네트워크 구성 변경 어려움, 회선 장애 시 전체 영향 |
| **메시(mesh)** | 그물망처럼 모든 노드가 연결 | 장애 내성 높음, 트래픽 분산 처리 가능 | 노드 추가 어려움, 구축/운용 비용 고가 |

### 보충 설명

**스푸핑**: LAN상에서 송신부의 패킷을 송신과 관련 없는 다른 호스트에 가지 않도록 하는 스위칭 기능을 마비시키거나 속여서 특정 노드에 해당 패킷이 오도록 처리하는 것
