## Q. TCP와 UDP의 차이점과 각각의 사용 사례를 설명하시오.

### 핵심 답변

| 구분 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 (3-way handshake) | 비연결 |
| 순서 보장 | O | X |
| 수신 여부 확인 | O | X |
| 패킷 교환 방식 | 가상회선 패킷 교환 | 데이터그램 패킷 교환 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 느림 | 빠름 |
| 사용 사례 | HTTP, 이메일, 파일 전송 | 스트리밍, DNS, 게임, VoIP |

### 보충 설명

**가상회선 패킷 교환 방식 (TCP)**:
- 각 패킷에 가상회선 식별자가 포함
- 모든 패킷을 전송하면 가상회선이 해제
- 패킷들은 전송된 **순서대로** 도착

**데이터그램 패킷 교환 방식 (UDP)**:
- 패킷이 독립적으로 이동하며 최적의 경로를 선택
- 하나의 메시지에서 분할된 여러 패킷이 서로 다른 경로로 전송될 수 있음
- 도착한 **순서가 다를 수** 있음

**TCP가 신뢰성이 있는 이유**: 3-way handshake 과정이 있어 연결 후 데이터 전송

**UDP가 신뢰성이 없는 이유**: 이 과정이 없음
