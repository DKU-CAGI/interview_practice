## Q. IPv4와 IPv6의 차이, 그리고 클래스 기반 할당 방식을 설명하시오.

### 핵심 답변

| 구분 | IPv4 | IPv6 |
|------|------|------|
| 비트 수 | 32비트 | 64비트 |
| 표기 | 8비트 단위로 점(.)으로 구분 | 16비트 단위로 점(:)으로 구분 |
| 예시 | 123.45.67.89 | 2001:db8::ff00:42:8329 |
| 현황 | 현재 가장 많이 사용 | 추세는 IPv6로 전환 중 |

### 보충 설명

**클래스 기반 할당 방식 (Classful Network Addressing):**

| 클래스 | 범위 | 구분 비트 | 용도 |
|--------|------|---------|------|
| A | 0.0.0.0 ~ 127.255.255.255 | 0 | 일반 통신 |
| B | 128.0.0.0 ~ 191.255.255.255 | 10 | 일반 통신 |
| C | 192.0.0.0 ~ 223.255.255.255 | 110 | 일반 통신 |
| D | - | - | 멀티캐스트 통신 |
| E | - | - | 예비용 |

**네트워크 주소와 브로드캐스트 주소:**
- 네트워크의 **첫 번째 주소** = 네트워크 구별 주소 (사용 불가)
- 네트워크의 **마지막 주소** = 브로드캐스트용 주소 (사용 불가)
- 예: 클래스 A로 12.0.0.0을 부여받으면, 12.0.0.1 ~ 12.255.255.254가 사용 가능한 호스트 주소

**단점**: 사용하는 주소보다 버리는 주소가 많음 → DHCP, IPv6, NAT로 해결
