## Q. ping, netstat, nslookup, tracert 명령어의 용도를 각각 설명하시오.

### 핵심 답변

| 명령어 | 용도 | 특징 |
|--------|------|------|
| **ping** | 네트워크 상태 확인 (대상 노드를 향해 일정 크기의 패킷 전송) | ICMP 프로토콜 사용, ICMP 차단된 경우 사용 불가 |
| **netstat** | 접속된 서비스들의 네트워크 상태 표시 | 네트워크 접속, 라우팅 테이블, 프로토콜 리스트 확인. 주로 포트 오픈 여부 확인 |
| **nslookup** | DNS 관련 내용 확인 | 특정 도메인에 매핑된 IP 확인 |
| **tracert** (Windows) / **traceroute** (Linux) | 목적지 노드까지 네트워크 경로 확인 | 어느 구간에서 응답 시간이 느려지는지 확인 가능 |

### 보충 설명

**네트워크 병목 현상 분석 명령어 활용:**
- 애플리케이션 코드에는 문제가 없지만 사용자가 서비스로부터 데이터를 가져오지 못할 때 사용
- 네트워크 관련 테스트와 네트워크와 무관한 테스트를 통해 '네트워크로부터 발생한 문제점'임을 확인 후 분석

**추가 도구:**
- **ftp**: 대형 파일을 전송하여 테스트
- **tcpdump**: 노드로 오고 가는 패킷을 캡처
- **wireshark**, **netmon**: 네트워크 분석 프로그램

**ping 사용 예시:**
```
ping www.google.com -n 12  (12번의 패킷 전송)
```
